# PLC磁通检测获取工序

Web 管理端在“工序规则管理”中选择 `PLC磁通检测获取`。界面显示名称固定为
`PLC磁通检测获取`，接口和数据库内部类型统一为 `plc_magnet_check`。

默认参数：

- PLC：`192.168.111.50`，Rack `0`，Slot `1`
- DB：`221`
- 原始块：从 Byte0 读取 26 字节
- 轮询：500ms
- 工艺等待超时：30秒

生产流程：

1. 写 `DBW0=1`并读回确认。
2. 等待`DBW2=1`。
3. 写`DBW4=1`并读回确认。
4. 等待`DBW6=1`。
5. 一次读取`DB221 Byte0~25`，本地解析左右磁通、极性和判定。
6. 仅当`DBW16=1`且`DBW24=1`时写`DBW8=1`，不再读回确认。
7. 上传结果到`plc_magnet_logs`，完成当前工序。

MES 只写整数1，不主动写0。DBW0、DBW4写入后需要读回确认；DBW8
写入后 PLC 会立即复位为0，因此客户端不回读。三个地址均由PLC负责复位。
任一写入读回失败、等待超时、块读取失败或左右判定NG时，当前工序
不会完成，也不会进入下一工序。
