# 产品实体、主条码切换与跨产线绑定

## 核心身份

- `product_instances.id` 是产品在系统内的永久身份。
- 条码保存在 `barcode_aliases`，条码变化不会改变产品实体 ID。
- `is_current=1` 的别名是当前生产主条码。
- 默认切换后旧码只用于追溯，不能继续生产。

## A 产线配置

1. 在项目管理维护 A 项目的物料编码和产品类型。
2. 在需要换码的任意工位新增 `主条码切换` 工序。
3. 默认配置为扫描旧码、扫描新码、新码设为当前主条码、禁用旧码。
4. 切换工位完成后，后续工位必须扫描新码。

## B 产线配置

1. 在项目管理维护 B 项目的物料编码和产品类型。
2. B 按自己的项目、工位和主条码独立生产。
3. B 各工位完成记录写入同一个 B 产品实体。

## A 绑定 B

1. 在 A 的装配工位新增 `子物料绑定` 工序。
2. 配置 B 项目、B 产品类型、需要数量和 B 必须完成的工位。
3. 操作员先扫描 A 当前主条码，再扫描 B 当前主条码。
4. B 未完成要求工位、类型不符或已绑定其他 A 时，系统拒绝绑定。

## 后续工位前置条件

在工位管理的“工位进入前置条件”中组合配置：

- 必须完成上一工位。
- 必须完成指定工位。
- 必须完成主条码切换。
- 必须绑定指定项目、类型和数量的子物料。
- 子物料必须完成指定工位。

桌面端在线生产时统一调用工位入口校验，不再只查询上一工位。
降级模式仍跳过服务器工艺依赖校验。

## 工艺路线配置

Web 管理端的“工艺路线配置”是原有“项目 → 工位 → 规则”的友好入口，
不会替换或批量改写已有工序。

- `route_name`、`route_order` 只负责路线分组和显示顺序。
- 生产放行读取 `station_dependencies.required_station_ids` 等显式依赖，
  不使用路线显示顺序推断 A/B 串行关系。
- A 主线和 B 子线可在同一项目中并行生产。
- 只有“子物料绑定”工序会写入 A-B 绑定关系，普通扫码不会产生绑定。
- “A主线绑定B子线路线”模板会生成 PLC A 起点、主条码切换、A-B 合并、
  A 后续和 B 两工位，并写入 B2→B1、A 合并→A 切换等显式依赖。

## 追溯

Web 后台“扫描记录查询”中的产品实体追溯支持输入：

- A 旧主条码。
- A 当前主条码。
- B 主条码。

查询结果包含条码别名、切换历史、工位完成历史以及父子物料绑定关系。

## 新系统数据库初始化

升级代码后重启 Web 服务，`init_db()` 会创建产品流相关表和配置字段：

- `product_instances`
- `barcode_aliases`
- `barcode_switch_records`
- `material_bindings`
- `station_dependencies`

系统尚未正式上线，不执行历史生产条码批量回填。正式启用前建议清理测试数据，
再依次配置项目物料类型、切换工序、绑定工序和后续工位前置条件。
